# FlowChartEditor 子系统

## 职责

FlowChartEditor 子系统负责流程图前端编辑器，包括导航、标签页、画布交互、连线校验、保存和与 WorkbookEditor 的跳转协作。

## 相关文档

1. [../WorkbookEditor/CompositeValueEditing.md](../WorkbookEditor/CompositeValueEditing.md)：定义复杂值的 JSON 兼容、类型树编辑和递归模态规则；流程图属性编辑后续应复用该机制。

## 当前已完成的工作

1. 已确定 FlowChartEditor 需要与 WorkbookEditor 并列存在，而不是作为表格编辑器内部的条件分支页面。
2. 已确定桌面端需要提供全局模式切换入口，用于在 Workbook 和 FlowChart 两种主编辑模式间切换。
3. 已确定从表格中的流程图字段进入编辑时，应打开流程图标签页，而不是把完整画布嵌入单元格。
4. 已确定流程图编辑器首版至少需要支持节点坐标持久化、端口连线和保存阻断式校验。
5. 已落地首版前端目录 `app/desktop/src/flowchart-editor/`，并与 `workbook-editor/` 保持并列拆分。
6. 已落地基于 React + SVG 的首版画布，实现节点拖拽、节点选择、端口连线和流程图保存。
7. 已落地左侧流程图文件树与节点库、中央画布、右侧 Inspector 的三段式编辑布局。
8. 已在节点 UI 中区分流程端口与计算端口，流程端口使用实线连线与方形端口，计算端口使用虚线连线与菱形端口。
9. 已支持在保存前执行本地结构校验，并阻止计算回路等显式错误写回工作区。
10. 已补齐高频画布编辑能力，包括缩放、框选、多选、批量拖拽、复制、剪切、粘贴和小地图。
11. 已补齐流程图模式下的快捷键编排，包括 Ctrl+A、Ctrl+C、Ctrl+X、Ctrl+V、Delete、Escape 和 Ctrl+S。

## 当前尚未实现的工作

1. 尚未实现跨编辑器共享标签页模型，当前流程图仍以单活动文件方式编辑。
2. 尚未实现从 WorkbookEditor 的流程图字段直接跳转并编排到统一标签体系。
3. 尚未实现流程图节点定义资产本身的专用编辑器，当前首版聚焦流程图实例编辑。
4. 尚未实现宿主侧正式保存校验器，当前阻断逻辑主要由前端本地校验承担。
5. 尚未实现更细粒度的批量属性编辑、对齐分布、自动排版等高级画布操作。
6. 尚未实现泛型节点的局部类型推断、显式 `typeArguments` 编辑与“未定型 / 二义性”语义诊断展示。
7. 尚未把复杂节点属性编辑统一到共享的复合值编辑机制上；后续 object / list / dictionary 等属性应复用 [../WorkbookEditor/CompositeValueEditing.md](../WorkbookEditor/CompositeValueEditing.md) 中的 JSON 兼容和递归模态规则。

## 交互边界

1. FlowChartEditor 只处理流程图视图和图编辑行为。
2. 节点定义的正式解析和保存协议应通过 DesktopHost 调用 Core，而不是在前端自行解释 json。
3. 与桌面壳共享的能力仅包括模式切换、通知、标签容器和宿主连接。
4. 对泛型标准节点，编辑器应尽量在本地完成类型参数求解与即时诊断，而不是把连线统一弱化成 `object` 并把错误全部推迟到代码编译期。
5. 对节点属性中的复杂值，编辑器应优先复用共享的复合值类型树和递归模态机制，而不是在 Inspector 中无限展开深层嵌套表单。

## 当前状态结论

FlowChartEditor 已完成首版可编辑落地，并已补齐一轮高频画布操作能力，与 DesktopApp 壳层形成了明确的目录边界。下一阶段除了跨编辑器标签编排和节点定义编辑器外，还应补齐泛型节点的本地类型推断、语义诊断与类型提示交互。

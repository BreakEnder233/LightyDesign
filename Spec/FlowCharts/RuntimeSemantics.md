# 流程图运行时语义

## 作用

本文件定义流程图运行时对象、生命周期、推理接口、暂停/结束语义以及异常传播规则。它为后续 Core 和 Generator 提供统一目标。

## 基本对象

### FlowChart 定义对象

1. FlowChart 定义对象描述一张流程图的静态结构。
2. 它持有节点、端口和连接关系，但不持有会变化的运行时上下文。
3. 同一张 FlowChart 定义可以同时创建任意多个 Flow 运行时对象。

### Flow 运行时对象

1. Flow 运行时对象由某个事件入口创建。
2. 每个 Flow 持有自己的 `context`。
3. 每个 Flow 持有自己的生命周期状态，因此可以独立重入、独立暂停、独立结束。
4. 同一张流程图允许并发存在多个 Flow 实例。

## 入口与重入

1. `event` 节点是合法的 Flow 创建入口。
2. 调用方可根据某个事件节点创建新的 Flow。
3. 因为所有运行时状态都由 Flow 持有，而不由 FlowChart 定义持有，所以可以任意重入同一张流程图，并创建任意多个 Flow。

## Flow 持有状态

首版 Flow 至少需要持有以下状态：

1. `context`
   运行时上下文，由业务侧提供。
2. `currentNodeId`
   当前所在流程节点 id。若尚未开始或已结束，可为空。
3. `isPaused`
   当前是否处于暂停状态。
4. `isCompleted`
   当前是否已经结束。

建议补充但非首版强制的状态：

1. `pauseReason`
   用于区分外部请求暂停、节点主动暂停或条件等待。
2. `lastTransition`
   用于调试和日志记录。

## 流程回路

1. 流程连接允许形成回路。
2. `if`、`while`、`for each`、`break`、`continue` 等控制节点依赖这一能力实现。
3. 允许流程回路不代表允许计算回路；计算连接仍然必须保持无环。

## 推理接口

首版 Flow 运行时应提供以下接口语义：

1. 推理 1 步
   执行一个最小流程推进单位。
2. 推理 N 步
   最多连续执行 N 个最小流程推进单位。
3. 推理直到结束
   持续执行，直到 `isCompleted == true`。
4. 推理直到暂停
   持续执行，直到 `isPaused == true` 或 `isCompleted == true`。
5. 推理直到条件满足
   持续执行，直到外部传入条件返回成立，或流程暂停，或流程结束。

## 步进语义

1. “一步”指一次可观察的流程推进，而不是单纯一次计算端口求值。
2. 一个流程节点在被调度执行后，可能：
   - 跳转到下一个流程节点。
   - 令当前 Flow 进入暂停状态。
   - 令当前 Flow 结束。
   - 抛出异常。
3. 计算端口求值属于该步内部行为，不单独暴露为 Flow 的公共步进单位。

## 暂停语义

Flow 的暂停来源分为两类：

1. 外部请求暂停
   调用方在运行接口的边界上要求 Flow 在合适时机停下。
2. 节点主动暂停
   某个流程节点在执行后要求 Flow 停止推进，直到后续再次被唤醒。

补充约定：

1. 当 `isPaused == true` 时，Flow 不应继续自动推进。
2. 被暂停的 Flow 可以在后续再次调用运行接口后继续推进。
3. 暂停不等于结束；`isPaused` 与 `isCompleted` 是两个独立状态。

## 结束语义

1. 当 Flow 到达显式结束节点或运行时定义的结束条件时，`isCompleted` 置为 `true`。
2. 已结束的 Flow 不应再继续推进。
3. 已结束的 Flow 可以保留 `context` 以供外部读取最终结果。

## 异常传播

1. 推理过程中产生的异常不需要在 Flow 内部持久保存。
2. 异常应直接抛给调用 Flow 接口的上层。
3. 首版不要求 Flow 记录异常历史。

## 中间值缓存

1. 首版不实现推理过程中的中间值缓存。
2. 若后续需要实现，中间值缓存应归 Flow 持有，而不是归 FlowChart 定义持有。
3. 缓存必须与具体 Flow 实例绑定，不能污染其它 Flow 实例。

## 代码生成建议

1. 每张流程图生成一个静态定义对象和一个 Flow 运行时类型。
2. 定义对象负责创建 Flow。
3. Flow 类型负责持有 `context` 和运行时状态，并暴露步进与运行接口。
4. 节点抛出的异常直接透传，不在生成骨架中吞掉。

## 当前状态结论

流程图运行时的核心语义已经明确：流程回路允许、Flow 可重入且可并发创建、上下文和运行态都归 Flow 持有、异常直接上抛、暂停与结束独立。后续主要工作是把这些语义映射到具体生成模板，而不是继续改文件格式。
